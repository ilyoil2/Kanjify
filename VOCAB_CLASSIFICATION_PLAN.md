# Kanjify Vocabulary Classification System Design

사용자가 단어장에서 단어를 효율적으로 분류하고 관리할 수 있는 맞춤형 시스템 설계안입니다.

## 1. 개요 (Overview)
학습자가 단어의 암기 상태나 중요도에 따라 단어를 특정 그룹(버튼)으로 분류하고, 그룹별로 설정된 기간 동안 단어장에서 숨기거나 영구 제외할 수 있는 기능을 제공합니다.

---

## 2. 주요 기능 (Core Features)
1.  **커스텀 버튼 시스템:** 사용자가 직접 버튼의 이름, 색상, 숨김 기간(일 단위 또는 영구)을 설정할 수 있습니다.
2.  **분류 및 자동 숨김:** 단어장에서 특정 그룹 버튼을 누르면 해당 기간만큼 목록에서 사라집니다.
3.  **그룹 관리 페이지:** 각 그룹에 어떤 단어들이 포함되어 있는지 확인하고, 필요시 즉시 복구(숨김 해제)할 수 있습니다.
4.  **영구 제외:** 특정 그룹(예: '영구 삭제' 버튼)은 복구 전까지 영구적으로 목록에서 제외됩니다.
5.  **기본 제공 버튼:** 회원가입 시 다음과 같은 버튼이 기본적으로 생성됩니다.
    *   **5일 숨기기** (5일 뒤 재노출)
    *   **10일 숨기기** (10일 뒤 재노출)
    *   **영구 숨기기** (사용자가 직접 복구하기 전까지 미노출)

---

## 3. 데이터베이스 설계 (DB Schema)

### 3.1 `tbl_word_button` (그룹 설정 테이블)
사용자가 정의하는 커스텀 버튼 정보를 저장합니다.
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | PK (Auto) | 버튼 식별자 |
| `name` | Char(50) | 버튼 이름 (예: "5일 뒤에", "영구 제외") |
| `color` | Char(20) | 버튼 색상 (Tailwind 컬러 코드 또는 Hex) |
| `hide_days` | Integer | 숨길 기간 (null이면 영구 숨김) |
| `created_at` | DateTime | 생성일 |

### 3.2 `tbl_word_status` (단어별 상태 관리 테이블)
개별 단어가 현재 어느 그룹에 속해 있고, 언제까지 숨겨지는지 기록합니다.
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `id` | PK (Auto) | 식별자 |
| `vocabulary` | FK (Vocabulary) | 대상 단어 |
| `button` | FK (Word Button) | 현재 적용된 버튼 그룹 |
| `hidden_until` | DateTime | 숨김 만료 시점 (null이면 영구 숨김) |
| `is_active` | Boolean | 현재 숨김 상태 여부 |
| `updated_at` | DateTime | 최근 상태 변경일 |

---

## 4. UI/UX 설계 (UI Design)

### 4.1 단어장 페이지 (Vocabulary Page)
-   단어 선택 시 상단에 사용자가 설정한 **`tbl_word_button` 리스트**가 동적으로 로드됩니다.
-   각 버튼 클릭 시 선택된 단어들이 해당 그룹의 로직에 따라 처리됩니다.

### 4.2 관리 페이지 (Archive/Group Management)
-   탭(Tab) 형식을 사용하여 각 그룹별 포함된 단어 리스트를 보여줍니다.
-   **"복구"** 버튼을 통해 숨김 기간이 남았더라도 다시 단어장으로 되돌릴 수 있습니다.

### 4.3 버튼 커스터마이징 (Settings)
-   버튼의 이름, 색상, 기간을 수정하거나 버튼 자체를 추가/삭제할 수 있는 설정 UI를 제공합니다.

---

## 5. 구현 로직 (Workflow)

1.  **목록 조회:** `tbl_vocabulary`를 조회할 때 `tbl_word_status`를 Join하여 `hidden_until`이 미래 시점이거나 영구 숨김 상태인 단어는 제외하고 가져옵니다.
2.  **분류 처리:** 사용자가 단어를 선택하고 버튼을 누르면, 백엔드에서 `hidden_until`을 `now() + hide_days`로 계산하여 `tbl_word_status`를 업데이트합니다.
3.  **동적 스케줄링:** 별도의 배치가 필요 없이, 조회 시점의 `timezone.now()`와 `hidden_until`을 비교하여 화면 노출 여부를 실시간 결정합니다.

---

## 6. 기대 효과
-   개인별 맞춤형 암기 주기 설정 가능 (에빙하우스 망각곡선 응용).
-   단어장 노이즈 제거를 통한 학습 집중도 향상.
-   체계적인 단어 관리 프로세스 구축.
